# Idea Bounty 部署指南

本指南面向低流量面试 MVP，采用单台 Linux 服务器：Nginx 托管前端并反向代理
FastAPI，Uvicorn 由 systemd 守护，PostgreSQL/pgvector 运行在 Docker 中。

```text
浏览器 --HTTPS--> Nginx :443 --/api--> Uvicorn :8000
                         `--/----> frontend/dist
                                      |
                               PostgreSQL :5432
                               （仅监听 127.0.0.1）
```

## 1. 服务器选择

当前域名未备案，建议先使用境外 `2 核 2 GB` 服务器部署 MVP。本项目的 AI 和
Embedding 都使用外部服务，低并发时该规格足够，Uvicorn 先使用一个 worker。

工信部规定在中国境内提供非经营性互联网信息服务需要履行备案手续，域名和仅 IP
访问都在适用范围内。因此不要把未备案站点部署到境内服务器规避备案。后续备案
完成后，再考虑迁移到境内 `4 核 4 GB` 服务器。参见[工信部备案管理办法](https://www.miit.gov.cn/gyhxxhb/jgsj/cyzcyfgs/bmgz/xxtxl/art/2024/art_84a0cfa0ebd049bbbe751dca9a008e56.html)。

以下假设服务器为 Ubuntu 24.04，域名为 `idea.zwenrenmuyuanzyj.me`，
项目目录为 `/opt/idea-bounty`。

## 2. 准备系统与代码

1. 将域名 `A` 记录指向服务器公网 IPv4。
2. 安全组/防火墙只对公网开放 `22`、`80` 和 `443`，不开放 `5432` 和 `8000`。
3. 安装 Git、Nginx、Docker、Certbot、rsync、Python 3.12/uv、Node.js 22.12+ 和 pnpm。

先安装 Docker 以外的基础软件：

```bash
sudo apt update
sudo apt install -y git nginx certbot python3-certbot-nginx rsync openssl curl
sudo systemctl enable --now nginx
```

再单独检查 Docker：

```bash
docker --version
```

- 如果命令成功，说明 Docker 已安装；不要再安装 Ubuntu 的 `docker.io`，直接启用服务。
- 如果命令不存在，按 [Docker 官方 Ubuntu 安装指南](https://docs.docker.com/engine/install/ubuntu/)
  配置 Docker 软件源并安装 Docker Engine。

Docker 官方源提供的 `containerd.io` 与 Ubuntu `docker.io` 依赖的 `containerd`
不能混装。确认 Docker 安装完成后执行：

```bash
sudo systemctl enable --now docker
sudo docker run --rm hello-world
```

将 uv 安装到系统 PATH 中，避免它只对当前 SSH 用户可见：

```bash
curl -LsSf https://astral.sh/uv/install.sh -o /tmp/uv-install.sh
sudo env UV_INSTALL_DIR=/usr/local/bin UV_NO_MODIFY_PATH=1 \
  sh /tmp/uv-install.sh
```

通过 NodeSource 安装 Node.js 22，再安装与项目开发环境一致的 pnpm：

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x \
  -o /tmp/nodesource_setup.sh
sudo -E bash /tmp/nodesource_setup.sh
sudo apt install -y nodejs
sudo npm install --global pnpm@11.11.0
```

确认版本：

```bash
docker --version
uv --version
node --version
npm --version
pnpm --version
nginx -v
```

创建专用系统用户并获取代码：

```bash
sudo useradd --system --user-group --create-home --home-dir /var/lib/idea-bounty \
  --shell /usr/sbin/nologin idea-bounty
sudo git clone https://github.com/Zwenrenmuyuan/IdeaBounty.git /opt/idea-bounty
sudo chown -R idea-bounty:idea-bounty /opt/idea-bounty
sudo -H -u idea-bounty uv --version
sudo -H -u idea-bounty node --version
sudo -H -u idea-bounty pnpm --version
sudo -H -u idea-bounty uv python install 3.12
```

如三条版本检查失败，说明工具只安装在当前管理员的个人目录；需改为系统级
PATH 可见的安装后再继续。`sudo -H` 会让 uv 的缓存和 Python 安装到
`/var/lib/idea-bounty`，而不是 SSH 用户的 HOME。

## 3. 启动生产数据库

根目录的 `compose.yaml` 包含开发账号、测试库和公开端口映射，只用于本地开发。
生产环境单独启动只监听回环地址的数据库容器。

先生成 URL 安全的随机密码，记录输出，后面的 `<db-password>` 都使用该值。
不要把真实密码发到聊天、工单或代码仓库中：

```bash
openssl rand -hex 24
sudo install -d -m 0750 /etc/idea-bounty
sudo vim /etc/idea-bounty/postgres.env
```

Vim 中按 `i` 进入编辑模式，粘贴内容后按 `Esc`，输入 `:wq` 并回车保存退出。

`postgres.env` 内容：

```dotenv
POSTGRES_DB=idea_bounty
POSTGRES_USER=idea_bounty
POSTGRES_PASSWORD=<db-password>
```

```bash
sudo chmod 600 /etc/idea-bounty/postgres.env
sudo docker volume create idea_bounty_postgres
sudo docker run -d \
  --name idea-bounty-db \
  --restart unless-stopped \
  --env-file /etc/idea-bounty/postgres.env \
  -p 127.0.0.1:5432:5432 \
  -v idea_bounty_postgres:/var/lib/postgresql/data \
  pgvector/pgvector:0.8.2-pg16-bookworm
sudo docker exec idea-bounty-db pg_isready -U idea_bounty -d idea_bounty
```

## 4. 配置并启动后端

创建 `/opt/idea-bounty/backend/.env`：

```bash
sudo vim /opt/idea-bounty/backend/.env
```

```dotenv
IDEA_BOUNTY_APP_NAME=idea-bounty-api
IDEA_BOUNTY_APP_ENV=production
IDEA_BOUNTY_DEBUG=false
IDEA_BOUNTY_DATABASE_URL=postgresql+psycopg://idea_bounty:<db-password>@127.0.0.1:5432/idea_bounty

AI_BASE_URL=https://provider.example.com/v1
AI_API_KEY=<ai-api-key>
AI_MODEL_ID=<chat-model-id>
AI_TIMEOUT_SECONDS=60
AI_MAX_RETRIES=2
AI_TEMPERATURE=0.2

EMBEDDING_BASE_URL=https://provider.example.com/v1
EMBEDDING_API_KEY=<embedding-api-key>
EMBEDDING_MODEL_ID=BAAI/bge-m3
EMBEDDING_DIMENSIONS=1024
EMBEDDING_TIMEOUT_SECONDS=60
EMBEDDING_MAX_RETRIES=2
```

```bash
sudo chown idea-bounty:idea-bounty /opt/idea-bounty/backend/.env
sudo chmod 600 /opt/idea-bounty/backend/.env
sudo -H -u idea-bounty /bin/bash -c \
  'cd /opt/idea-bounty/backend && uv sync --frozen --no-dev'
sudo -H -u idea-bounty /bin/bash -c \
  'cd /opt/idea-bounty/backend && uv run --no-sync alembic upgrade head'
```

新建 `/etc/systemd/system/idea-bounty.service`：

```bash
sudo vim /etc/systemd/system/idea-bounty.service
```

```ini
[Unit]
Description=Idea Bounty FastAPI
After=network-online.target docker.service
Wants=network-online.target
Requires=docker.service

[Service]
Type=simple
User=idea-bounty
Group=idea-bounty
WorkingDirectory=/opt/idea-bounty/backend
EnvironmentFile=/opt/idea-bounty/backend/.env
Environment=PYTHONUNBUFFERED=1
ExecStart=/opt/idea-bounty/backend/.venv/bin/uvicorn idea_bounty.main:app \
  --host 127.0.0.1 --port 8000 --workers 1 \
  --proxy-headers --forwarded-allow-ips=127.0.0.1
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now idea-bounty
curl http://127.0.0.1:8000/api/health
```

应返回 `{"status":"ok","service":"idea-bounty-api"}`。

## 5. 构建前端与配置 Nginx

前端使用相对路径 `/api`，与后端同域部署时不需要 CORS 或额外构建变量。

```bash
sudo -H -u idea-bounty /bin/bash -c \
  'cd /opt/idea-bounty/frontend && pnpm install --frozen-lockfile && pnpm build'
sudo install -d -o www-data -g www-data /var/www/idea-bounty
sudo rsync -a --delete /opt/idea-bounty/frontend/dist/ /var/www/idea-bounty/
```

新建 `/etc/nginx/sites-available/idea-bounty`：

```bash
sudo vim /etc/nginx/sites-available/idea-bounty
```

```nginx
server {
    listen 80;
    listen [::]:80;
    server_name idea.zwenrenmuyuanzyj.me;

    root /var/www/idea-bounty;
    index index.html;
    client_max_body_size 1m;

    location /api/ {
        proxy_pass http://127.0.0.1:8000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_connect_timeout 5s;
        proxy_read_timeout 600s;
        proxy_send_timeout 600s;
    }

    location /assets/ {
        try_files $uri =404;
        expires 7d;
        add_header Cache-Control "public, immutable";
    }

    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

`600s` 超时是为当前同步 AI 评估、Embedding 和查重流水线预留时间。

```bash
sudo ln -s /etc/nginx/sites-available/idea-bounty /etc/nginx/sites-enabled/idea-bounty
sudo rm -f /etc/nginx/sites-enabled/default
sudo nginx -t
sudo systemctl reload nginx
```

## 6. 启用 HTTPS

`IDEA_BOUNTY_APP_ENV=production` 会让会话 Cookie 带上 `Secure`，因此正常登录前必须完成
HTTPS 配置。确认域名已解析且 80/443 端口可访问后执行：

```bash
sudo certbot --nginx -d idea.zwenrenmuyuanzyj.me
sudo certbot renew --dry-run
```

Certbot 询问时选择将 HTTP 重定向到 HTTPS。

验收：

```bash
curl https://idea.zwenrenmuyuanzyj.me/api/health
curl -I https://idea.zwenrenmuyuanzyj.me/
```

浏览器中再完成注册、登录、投稿和后台页面的桌面端/移动端检查。

## 7. 初始化管理员

先通过页面注册一个普通账号，再在服务器执行：

```bash
sudo -H -u idea-bounty /bin/bash -c \
  'cd /opt/idea-bounty/backend && uv run --no-sync python scripts/promote_admin.py <username>'
```

重新登录后即可访问 `/admin`。管理员仅用于审核，不能投稿。打款功能只产生
`SIM-` 开头的模拟流水，不会发起真实支付。

## 8. 更新、备份与回滚

每次更新前备份数据库：

```bash
sudo install -d -m 0700 /var/backups/idea-bounty
sudo sh -c 'docker exec idea-bounty-db pg_dump \
  -U idea_bounty -d idea_bounty -Fc \
  > /var/backups/idea-bounty/idea_bounty_$(date +%F_%H%M%S).dump'
```

记录当前 Git commit，然后更新：

```bash
cd /opt/idea-bounty
sudo -H -u idea-bounty git pull --ff-only
sudo -H -u idea-bounty /bin/bash -c \
  'cd backend && uv sync --frozen --no-dev && uv run --no-sync alembic upgrade head'
sudo -H -u idea-bounty /bin/bash -c \
  'cd frontend && pnpm install --frozen-lockfile && pnpm build'
sudo rsync -a --delete frontend/dist/ /var/www/idea-bounty/
sudo systemctl restart idea-bounty
sudo nginx -t && sudo systemctl reload nginx
```

如需回滚，先停止后端，恢复更新前的 Git commit 和数据库备份，再重新构建。不要只回退
代码而保留不兼容的新数据库结构。

## 9. 排障命令

```bash
sudo systemctl status idea-bounty
sudo journalctl -u idea-bounty -n 200 --no-pager
sudo nginx -t
sudo tail -n 200 /var/log/nginx/error.log
sudo docker ps
sudo docker logs --tail 200 idea-bounty-db
sudo docker exec idea-bounty-db pg_isready -U idea_bounty -d idea_bounty
curl http://127.0.0.1:8000/api/health
```

常见问题：

- 可打开页面但无法登录：检查 HTTPS、Cookie 和 Nginx `/api/` 代理。
- 投稿进入 `failed`：检查 AI/Embedding 配置和 systemd 日志，不要把密钥粘贴到工单或提交中。
- 数据库连接失败：确认容器健康、密码一致且 URL 使用 `127.0.0.1`。
- 前端路由刷新 404：检查 Nginx 是否保留 `try_files ... /index.html`。
